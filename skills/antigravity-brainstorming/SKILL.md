# Directive: Antigravity Brainstorming

> Use this skill only when normal planning is blocked and the team needs bounded lateral options, not more generic brainstorming.

## 🎯 MISSION (CORE OBJECTIVES)
1. **Lateral Matrix Ideation:** Connect disparate domain paradigms (e.g., Aerospace telemetry concepts applied to React render cycles) to formulate paradigm-shifting solutions.
2. **First-Principles Dissection:** Deconstruct blockers down to their foundational atomic truths (Memory limitation, Network Physics, Time Complexity) instead of patching symptoms.
3. **Scope Discipline:** Do not expand tooling or architecture unless the idea survives product, engineering, and verification scrutiny.

## ⚙️ EXECUTION PIPELINE (THE IDEATION CYCLE)

### Phase 1: Blockage Ingestion
- Read the root `agents.md`, the active feature docs, and the blocker evidence first.
- Do not brainstorm around a vague feeling of "stuck"; identify the actual constraint.

### Phase 2: Option Shaping
- Generate three options at most:
  - low-risk path
  - medium-risk acceleration
  - high-risk structural change
- Each option must include expected upside, blast radius, and verification burden.

### Phase 3: Brainstorming Emission
- Convert the surviving option into spec-ready input: scope, assumptions, risks, and docs impact.
- If no option can be verified cheaply enough, say so and recommend staying with the simpler path.

## 🛡️ MANDATORY PROTOCOLS (ENTERPRISE BOUNDARIES)
### Protocol 1: Physical Grounding
- Ideas without an execution path are noise. End with concrete next steps that a planner can route into `spec.md`, `plan.md`, or `tasks.md`.
- **[REPORT]**: Emitted upon expanding the consciousness scope of the Engineering Master Matrix.
