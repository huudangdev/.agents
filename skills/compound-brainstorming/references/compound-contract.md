# Compound Brainstorm Contract

Use this skill to pressure-test a feature across product, UX, engineering, and security.

## Required Inputs

- The active spec and constraints
- Product value, implementation cost, risk, and verification burden
- The current feature boundary

## Decision Rules

- Score ideas against value, cost, risk, and verification burden.
- Kill ideas that add wide blast radius for weak gain.
- Stay inside the current feature boundary.

## Output Contract

- Present the cross-functional matrix.
- State which option survives and why.
- End with actionable PM or engineering inputs.
