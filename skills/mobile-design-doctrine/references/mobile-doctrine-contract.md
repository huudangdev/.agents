# Mobile Doctrine Contract

Use this reference when `mobile-design-doctrine` needs to produce mobile UX
guidance.

## 1. Flow

- Screen.
- Gesture.
- State transition.

## 2. Constraints

- Thumb reach.
- Safe area.
- Accessibility.
- Performance.

## 3. Recovery

- Failure state.
- Rollback or retry.
- User-visible feedback.

## 4. Output

End with:

- mobile recommendation
- risks
- next design step
