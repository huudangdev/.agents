# Simulator Contract

Use this reference when `qa-simulator` needs to produce live UI evidence or a
route verdict.

## 1. Scope

- Route or screen.
- Device or browser assumption.

## 2. Checks

- Render.
- Interaction.
- State change.
- Failure state.

## 3. Evidence

- Command or tool used.
- Result.
- Visible issue, if any.

## 4. Output

End with:

- route verdict
- blocker or residual risk
- next action
