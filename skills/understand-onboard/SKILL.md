# /understand-onboard

Generate a comprehensive onboarding guide from the project's knowledge graph.

## Required Reads

- [knowledge-graph-contract.md](references/knowledge-graph-contract.md)

## Operating Rules

- Read project metadata, layers, and tour.
- Read file-level nodes only.
- Identify complexity hotspots.
- Keep the guide high-level and practical.

## Output Expectations

- Cover project overview, architecture layers, key concepts, guided tour, file map, and complexity hotspots.
- Offer to save the guide to `docs/ONBOARDING.md`.
- Suggest the user commit it to the repo for the team.
