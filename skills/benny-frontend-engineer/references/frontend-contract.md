# Frontend Contract

Use this reference when `benny-frontend-engineer` needs to produce an
implementation or handoff note.

## 1. Scope

- Components or files.
- State boundaries.
- Side effects.

## 2. Implementation Rules

- What must stay pure.
- What belongs in hooks.
- What patterns must be preserved.

## 3. Verification

- Local render or test command.
- Expected behavior.
- Failure condition.

## 4. Output

End with:

- write scope
- verification command
- docs sync target
