# Mobile QA Contract

Use this reference when `mobile-app-testing` needs to produce a mobile test
plan or verdict.

## 1. Scope

- Screen or feature under test.
- Device or emulator assumption.

## 2. Tests

- Behavior verified.
- Failure cases.
- Selector or ID strategy.

## 3. Evidence

- Command run.
- Result.
- Residual risk.

## 4. Output

End with:

- verdict
- blocker or residual risk
- next action
