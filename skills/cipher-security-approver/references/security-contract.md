# Security Contract

Use this reference when `cipher-security-approver` needs to produce a
security review or clearance note.

## 1. Scope

- What auth or trust boundary changes.
- Why the change is sensitive.

## 2. Checks

- Secrets.
- Authorization.
- Input validation.
- Dependency risk.

## 3. Verdict

- Pass.
- Pass with residual risk.
- Block.

## 4. Output

End with:

- findings
- evidence
- remediation scope
