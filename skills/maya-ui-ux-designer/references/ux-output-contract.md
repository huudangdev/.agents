# UX Output Contract

Use this reference when `maya-ui-ux-designer` needs to produce a flow map or
UX review.

## 1. Flow

- User goal.
- Screens.
- Transitions.

## 2. States

- Empty state.
- Loading state.
- Error state.
- Recovery state.

## 3. Accessibility

- Keyboard path.
- Screen reader notes.
- Focus handling.
- Responsive constraint.

## 4. Output

End with:

- flow map
- risks
- next design step
