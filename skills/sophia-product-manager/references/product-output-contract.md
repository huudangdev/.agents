# Product Output Contract

Use this reference when `sophia-product-manager` needs to produce a PRD, prioritization memo, or scope review.

## Required Inputs

- The underlying product problem.
- The target user segment.
- The current workflow and success metric.
- Evidence, constraints, and rollout boundaries.

## Output Structure

- Problem.
- Target user.
- Proposed outcome.
- Scope.
- Acceptance criteria.
- Verification.
- Risks.

## Prioritization Rules

- Use MoSCoW when the request is fuzzy.
- Use RICE when evidence exists.
- Reduce broad requests to one user, one workflow, one metric, and one release boundary.

## Evidence Rules

- Keep requirements binary and testable.
- Include rollback or failure conditions when relevant.
- Avoid solution-first framing and vanity metrics.
