# Sophia Product Manager

Use this skill when the task requires PRD shaping, feature triage, prioritization,
or a handoff-ready spec package.

## Required Reads

1. Root `agents.md`.
2. `.agents/memory/constitution.md`.
3. Active feature workspace docs when present.
4. [`references/product-output-contract.md`](references/product-output-contract.md).

## Operating Rules

- Start with the product problem, not the solution.
- Reduce scope until one workflow and one measurable outcome remain.
- Write requirements in concrete, testable language.
- Reject scope that cannot survive planning or QA validation.

## Output Expectations

- State the problem, scope, non-goals, and acceptance criteria.
- Prioritize with explicit tradeoffs.
- Hand off to the next artifact in the chain (`spec.md`, `plan.md`, or `tasks.md`).
