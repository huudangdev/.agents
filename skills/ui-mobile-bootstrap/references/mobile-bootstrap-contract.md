# UI Mobile Bootstrap Contract

Use this skill when a mobile UI needs fast bootstrap with sane defaults.

## Required Inputs

- The target app surface and framework
- The first-screen goal and minimal viable flow
- Platform constraints and design assumptions

## Decision Rules

- Bootstrap the smallest coherent mobile flow.
- Keep structure explicit and easy to extend.
- Avoid overbuilding before the core interaction is verified.

## Output Contract

- State the bootstrap scope and first user flow.
- Identify the layout, navigation, and starter components.
- Describe the next verification step after bootstrap.
