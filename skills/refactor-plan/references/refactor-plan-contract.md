# Refactor Plan Contract

Use this reference when `refactor-plan` needs to produce an incremental
refactor blueprint.

## 1. Scope

- What code path is being changed.
- What must remain stable.

## 2. Phases

- Phase name.
- Goal.
- Write scope.
- Verification command.
- Docs sync target.

## 3. Risk

- Regression risk.
- Dependency collision risk.
- Rollback or stop condition.

## 4. Output

End with:

- ordered phases
- blockers
- verification
