# Development Ledger Architect

Use this skill when creating, validating, migrating, or auditing development docs.

## Required Reads

- [development-ledger-contract.md](references/development-ledger-contract.md)
- The current development manifest and index when they exist.

## Operating Rules

- Use epic-first topology and canonical IDs.
- Keep docs specific, visual, and traceable.
- Reconcile stale or template-only docs before implementation proceeds.

## Output Expectations

- State what docs must change and why.
- Identify the owning epic and related artifacts.
- Describe the validation commands that must pass.
