# Implementation Contract

Use this reference when `alan-tech-lead` needs to produce architecture or
execution handoff guidance.

## 1. Architecture Summary

- What is being built.
- What system boundary changes.
- What dependencies are affected.

## 2. Execution Constraints

- Required inputs.
- Allowed outputs.
- Blocking conditions.

## 3. Design Decisions

- Chosen approach.
- Rejected alternatives.
- Why the chosen approach fits the current package.

## 4. Verification Hooks

- How the design will be tested.
- What evidence is required.
- What failure condition blocks implementation.

## 5. Handoff Checklist

- Scope is validated.
- Design boundaries are explicit.
- Rollback or failure path is stated.
- The next implementer can act without guessing.
