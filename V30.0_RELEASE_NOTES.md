# Marcus Fleet V30.0 Release Notes

## Focus

V30.0 upgrades Marcus Fleet from a strong multi-agent execution framework into a
more governable planning and research system. The release is additive: existing
slash commands and legacy `/planning` output files remain valid.

## Major Changes

- Added Spec Kit-inspired feature lifecycle:
  - `.agents/memory/constitution.md`
  - `.agents/specs/<feature-id>/`
  - `.agents/templates/*`
  - `.agents/scripts/create_feature_spec.py`
  - `.agents/scripts/validate_specs.py`
  - `/marcus.specify`, `/marcus.clarify`, `/marcus.plan`, `/marcus.tasks`, `/marcus.verify`
- Upgraded `/planning` to V30 deep research:
  - Preserves eight legacy `/docs` outputs.
  - Adds `/docs/research/sources.jsonl`, `evidence.jsonl`, `claims.jsonl`,
    `contradictions.md`, and `research_manifest.json`.
  - Adds claim/evidence/contradiction validation through
    `.agents/scripts/validate_planning_research.py`.
- Hardened sandbox execution:
  - Removed `eval`.
  - Rejects shell metacharacter chaining.
  - Uses an explicit allow-list for bounded commands.
- Unified TrustGraph runtime configuration:
  - `.agents/trustgraph.env.example`
  - `.agents/adapters/trustgraph_config.py`
  - Shared Neo4j/Chroma settings across adapters and viewer API.
- Improved TrustGraph viewer:
  - Added `/api/health`.
  - Added live runtime status UI.
  - Removed explicit `any` typing from key graph components.
  - Replaced shell string execution in Chroma route with `execFile`.
- Updated CI template:
  - Adds spec validation.
  - Runs planning research validation when `/docs/research/` exists.
  - Calls incremental TrustGraph sync with explicit changed files.

## V30.1 Addendum: Develop Knowledge Ledger

- Upgraded `/develop` so code-phase agents must create implementation memory,
  not only source code.
- Added `/docs/development/` artifact contract:
  - `development_manifest.json`
  - `index.md`
  - `epics/*.md`
  - `modules/*.md`
  - `features/*.md`
  - `pages/*.md`
  - `tasks/*.md`
- Added templates for each development artifact bucket.
- Added `.agents/scripts/create_development_docs.py` to scaffold the code-phase
  documentation package.
- Added `.agents/scripts/validate_development_docs.py` to block incomplete
  code-phase documentation.
- Updated `.clinerules`, `USAGE_GUIDE.md`, and `/develop` so implementation
  notes must include `owner_skill`, `source_trace`, write/code scope, and
  verification evidence.

## V30.2 Addendum: Continuous Documentation Sync

- Added a PM-grade continuity gate for long POC builds where `/develop` repeats
  many code/test cycles.
- Added `/docs/development/sync/` as an append-only documentation sync ledger.
- Development scaffolding creates `sync_manifest.json`; individual sync notes are
  created only when a real code slice changes source files.
- Added `.agents/scripts/create_doc_sync_note.py` to create per-code-slice sync
  notes from explicit changed files or `git diff`.
- Added `.agents/scripts/validate_doc_sync.py` to fail when source files changed
  without a sync note and documentation review.
- Updated `/develop` with Node 6.5 Doc Sync Checkpoint.
- Updated `/quick_fix` so behavior-changing hotfixes also create sync notes and
  run doc-sync validation.
- Updated `.clinerules`, README, usage guide, CI template, and enterprise
  architecture so docs are updated by append or targeted patch, not wholesale
  replacement.

## V30.3 Addendum: Substantive Documentation Quality

- Added `.agents/DEVELOPMENT_DOCS_QUALITY_RUBRIC.md`.
- Strengthened `validate_development_docs.py` to reject template residue,
  unchecked boxes, `TBD`, `pending`, shallow notes, missing rationale, and
  missing concrete code paths.
- Strengthened `validate_doc_sync.py` to reject shallow sync notes and generic
  unfinished decisions.
- Upgraded development templates with PM notes, implementation commentary,
  change logs, rationale, evidence, risk, and quality-bar instructions.

## Compatibility

- `/planning` still writes:
  - `/docs/prd.md`
  - `/docs/tasks.md`
  - `/docs/knowledge.md`
  - `/docs/decisions.md`
  - `/docs/memory.md`
  - `/docs/planning/flows.md`
  - `/docs/planning/screens.md`
  - `/docs/planning/diagrams.md`
- `/design` and `/develop` continue to use approved `/docs` artifacts.
- `/quick_fix` remains the correct route for small localized fixes.
- TrustGraph adapters still degrade gracefully when Neo4j/Chroma are offline.

## Recommended Validation

```bash
python3 .agents/scripts/validate_specs.py
python3 .agents/scripts/validate_planning_research.py --root .
python3 .agents/scripts/validate_development_docs.py --strict-counts
python3 .agents/scripts/validate_doc_sync.py --strict
cd .agents/trustgraph-viewer && npm run lint && npm run build
```
