# Marcus Fleet V30.0 Release Notes

## Focus

V30.0 upgrades Marcus Fleet from a strong multi-agent execution framework into a
more governable planning and research system. The release is additive: existing
slash commands and legacy `/planning` output files remain valid.

## Major Changes

- Added Spec Kit-inspired feature lifecycle:
  - `.agents/memory/constitution.md`
  - `.agents/specs/<feature-id>/`
  - `.agents/templates/*`
  - `.agents/scripts/create_feature_spec.py`
  - `.agents/scripts/validate_specs.py`
  - `/marcus.specify`, `/marcus.clarify`, `/marcus.plan`, `/marcus.tasks`, `/marcus.verify`
- Upgraded `/planning` to V30 deep research:
  - Preserves eight legacy `/docs` outputs.
  - Adds `/docs/research/sources.jsonl`, `evidence.jsonl`, `claims.jsonl`,
    `contradictions.md`, and `research_manifest.json`.
  - Adds claim/evidence/contradiction validation through
    `.agents/scripts/validate_planning_research.py`.
- Hardened sandbox execution:
  - Removed `eval`.
  - Rejects shell metacharacter chaining.
  - Uses an explicit allow-list for bounded commands.
- Unified TrustGraph runtime configuration:
  - `.agents/trustgraph.env.example`
  - `.agents/adapters/trustgraph_config.py`
  - Shared Neo4j/Chroma settings across adapters and viewer API.
- Improved TrustGraph viewer:
  - Added `/api/health`.
  - Added live runtime status UI.
  - Removed explicit `any` typing from key graph components.
  - Replaced shell string execution in Chroma route with `execFile`.
- Updated CI template:
  - Adds spec validation.
  - Runs planning research validation when `/docs/research/` exists.
  - Calls incremental TrustGraph sync with explicit changed files.

## Compatibility

- `/planning` still writes:
  - `/docs/prd.md`
  - `/docs/tasks.md`
  - `/docs/knowledge.md`
  - `/docs/decisions.md`
  - `/docs/memory.md`
  - `/docs/planning/flows.md`
  - `/docs/planning/screens.md`
  - `/docs/planning/diagrams.md`
- `/design` and `/develop` continue to use approved `/docs` artifacts.
- `/quick_fix` remains the correct route for small localized fixes.
- TrustGraph adapters still degrade gracefully when Neo4j/Chroma are offline.

## Recommended Validation

```bash
python3 .agents/scripts/validate_specs.py
python3 .agents/scripts/validate_planning_research.py --docs docs
cd .agents/trustgraph-viewer && npm run lint && npm run build
```
